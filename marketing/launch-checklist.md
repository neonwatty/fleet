# Launch checklist

## Before merge

- Review `site/index.html` at desktop and mobile widths.
- Confirm the GitHub Pages workflow passes on the PR.
- Confirm repository Pages settings use GitHub Actions as the source.
- Confirm the website URL should be `https://neonwatty.github.io/fleet/`.
- Decide whether starter mock visuals are acceptable for the first public pass.

## Before promotion

- Publish or verify a current GitHub release.
- Open `https://github.com/neonwatty/fleet/releases/latest` and confirm the release assets are present.
- Open the deployed Pages URL and verify the release, GitHub, and README links.
- Check social preview rendering with the deployed Open Graph image URL.
- Pick one primary launch channel first, then adapt the copy for others.

## Follow-up assets

- Capture a real TUI screenshot.
- Capture a native menu bar popover screenshot.
- Record a short launch-to-tunnel demo.
- Replace the starter social card if the real product screenshots are stronger.

See `marketing/screenshot-guide.md` for the capture plan.
