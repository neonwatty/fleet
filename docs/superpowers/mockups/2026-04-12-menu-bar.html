Fleet — menu bar mockup

Visual mockup of the SwiftBar-hosted fleet indicator. The top strip simulates the macOS menu bar; the panel below simulates the native NSMenu dropdown that SwiftBar renders from the plugin script's output. Everything here is what the user sees — the actual implementation is just a shell script calling fleet status --json.

Menu bar title 3/4 · 2 CC = 3 machines online of 4 total · 2 live Claude Code instances across the fleet. Dropdown shows one machine per section with the session labels indented underneath. Live labels use a filled bullet; stale labels (no matching CC process) use an outlined bullet and muted text. Accounts render in accent color next to the machine name. Offline machines keep their last-known label so you can still see "mm3 was running bugdrop" after a restart.

Rendering notes

● 3/4 · 2 CC — menu bar title format. Left of · is online/total machines; right is live CC count.
⚠ 4/4 · 5 CC — warning prefix when any machine's score label ≥ busy. Color shifts from default → amber → red based on worst-case machine.
Filled bullet = live label (matching CC process). Outlined bullet = stale label (last-known, no matching process).
Swap coloring — default while swap is low, amber once swap crosses ~1GB (early pressure signal), red once it crosses ~4GB (actively thrashing). Thresholds are configurable in config.toml; defaults match fleet's existing stress_threshold heuristics.
Dropdown is a native NSMenu rendered by SwiftBar — this HTML mockup approximates its appearance but the real thing uses native vibrancy, system font, and macOS menu row metrics. Hover states, keyboard navigation, and dismissal are all handled by AppKit.
